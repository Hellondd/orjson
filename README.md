# Fast Telemetry Dumper (orjson)

Высокопроизводительный скрипт для сериализации и десериализации телеметрических данных. Проект демонстрирует расширенные возможности библиотеки `orjson` для работы с нетипичными и сложными структурами данных в Python.

## Использование возможностей `orjson`

Проект не ограничивается базовым `dumps`/`loads` и задействует следующий функционал:

1. **`orjson.Fragment`**: Внедрение сырых байтовых JSON-строк напрямую в структуру без накладных расходов на их десериализацию и повторную сериализацию. Ускоряет работу с кешированными данными.
2. **Параметр `default`**: Интеграция кастомного энкодера `custom_encoder` для обработки пользовательских классов (`FinancialMetric`) и типов из стандартной библиотеки, не поддерживаемых "из коробки" (`decimal.Decimal`).
3. **Нативная поддержка типов**: Обработка `uuid.UUID`, `datetime.datetime` и экземпляров `dataclasses.dataclass` на уровне C-кода `orjson` без написания дополнительных парсеров.
4. **Битовые флаги (`option`)**:
   * `orjson.OPT_INDENT_2` — форматирование вывода.
   * `orjson.OPT_SORT_KEYS` — детерминированный порядок ключей.
   * `orjson.OPT_NON_STR_KEYS` — принудительная конвертация целочисленных ключей словаря (например, HTTP-статусов) в строки, предотвращающая `TypeError`.
   * `orjson.OPT_OMIT_MICROSECONDS` — стандартизация временных меток.
   * `orjson.OPT_APPEND_NEWLINE` — корректное завершение файла для POSIX-систем.
